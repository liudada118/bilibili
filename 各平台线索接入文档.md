## 1. 微信客服

### 接入方式

**被动回调模式**：企业微信客服平台主动将客户消息推送到本系统的回调地址。系统接收消息后自动创建客户记录（来源标记为 `wechat`）。

### 后台配置项

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `corpId` | 必填 | 企业微信的企业ID |
| `kfSecret` | 必填 | 微信客服应用的Secret |
| `token` | 必填（回调模式） | 回调URL验证用的Token |
| `encodingAesKey` | 必填（回调模式） | 消息加解密密钥 |
| `proxyUrl` | 可选 | HTTP代理地址（用于访问微信API） |

### Webhook 地址

```
GET/POST https://kefu.jq-industries.com/api/wechat/callback
```

- **GET**：微信后台配置回调URL时的验证请求，参数包含 `msg_signature`、`timestamp`、`nonce`、`echostr`
- **POST**：接收消息推送，Body 为 XML 格式（AES加密），包含 `<Encrypt>` 字段

### 配置步骤

1. 在系统"设置 → 平台消息 → 微信客服"中填写 `corpId`、`kfSecret`、`token`、`encodingAesKey`
2. 在企业微信管理后台 → 客服 → 开发配置中，将回调URL设为 `https://kefu.jq-industries.com/api/wechat/callback`
3. 填入相同的 Token 和 EncodingAESKey，验证通过后即可接收消息

### 线索创建逻辑

- 当客户发送消息或进入客服会话时，系统自动创建客户记录
- `externalUserId` = 微信客服的外部用户ID
- `source` = `"wechat"`
- `status` = `"new_lead"`

## 2. 小红书 / 抖音（美洽中转）

### 接入方式

**Webhook 推送模式**：美洽客服系统将小红书/抖音渠道的对话推送到本系统。系统根据对话的 `mainChannel` 字段区分来源（小红书 → `xiaohongshu_meiqia`，抖音 → `douyin_meiqia`）。

### 后台配置项

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `accessToken` | 必填 | 美洽API的Access Token |
| `webhookSecret` | 可选 | Webhook签名验证密钥 |
| `webhookUrl` | 可选 | 系统自动填入的Webhook地址 |

### Webhook 地址

```
POST https://kefu.jq-industries.com/api/meiqia/webhook
```

### 请求格式

美洽标准 Webhook 格式，Body 为 JSON：

```json
{
  "event": "conversation.created",
  "data": {
    "conversation": {
      "id": "conv_12345",
      "main_channel": "xiaohongshu",
      "client_info": {
        "id": "client_001",
        "name": "用户昵称"
      },
      "agent_info": {
        "id": "agent_001",
        "name": "客服小张"
      },
      "status": "open",
      "summary": "咨询产品价格",
      "conv_content": [
        {
          "type": "client",
          "content": "你好，想了解一下产品",
          "created_at": "2026-04-28 16:30:00"
        }
      ]
    }
  }
}
```

### 签名验证

请求头 `meiqia-signature` 包含签名，算法为 `SHA1(body_string + webhookSecret)`。

### 配置步骤

1. 在系统"设置 → 平台消息 → 美洽"中填写 Access Token 和 Webhook Secret
2. 在美洽后台 → 设置 → Webhook 中配置推送地址为 `https://kefu.jq-industries.com/api/meiqia/webhook`
3. 确保美洽中已绑定小红书/抖音渠道

## 3. B站私信

### 接入方式

**Webhook 推送模式**：通过自建中间件或B站开放平台将私信消息推送到本系统。

### 后台配置项

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `token` | 可选 | 签名验证Token |
| `webhookUrl` | 可选 | 系统Webhook地址 |
| `appKey` | 可选 | B站开放平台AppKey |
| `appSecret` | 可选 | B站开放平台AppSecret |

### Webhook 地址

```
POST https://kefu.jq-industries.com/api/bilibili/webhook
```

### 请求格式

Body 为 JSON：

```json
{
  "msg_id": "msg_001",
  "sender_uid": "12345678",
  "sender_name": "B站用户昵称",
  "receiver_uid": "87654321",
  "msg_type": "text",
  "content": "你好，想了解你们的产品",
  "timestamp": 1714300200000
}
```

### 签名验证

请求头 `x-bili-signature` 包含签名，算法为 `HMAC-SHA256(body, token)`。

### 线索创建逻辑

- `externalUserId` = `bili_{sender_uid}`
- `source` = `"bilibili"`
- `status` = `"new_lead"`

## 4. 抖音私信（直连）

### 接入方式

**Webhook 推送模式**：抖音开放平台直接将私信事件推送到本系统。需要在抖音开放平台创建应用并配置 Webhook。

### 后台配置项

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `clientKey` | 必填 | 抖音开放平台应用的Client Key |
| `clientSecret` | 必填 | 抖音开放平台应用的Client Secret |
| `token` | 可选 | Webhook验证Token |
| `webhookUrl` | 可选 | 系统Webhook地址 |

### Webhook 地址

```
POST https://kefu.jq-industries.com/api/douyin/webhook
```

### 请求格式

抖音开放平台标准事件推送格式：

```json
{
  "event": "im_receive_msg",
  "from_user_id": "open_id_xxx",
  "to_user_id": "open_id_yyy",
  "content": {
    "message_type": "text",
    "text": "你好，想咨询一下",
    "conversation_short_id": "conv_123",
    "server_message_id": "msg_456",
    "create_time": 1714300200000,
    "user_infos": [
      {
        "open_id": "open_id_xxx",
        "nick_name": "抖音用户"
      }
    ]
  }
}
```

### Webhook 验证

首次配置时，抖音会发送验证请求：

```json
{
  "event": "verify_webhook",
  "content": {
    "challenge": "random_string_xxx"
  }
}
```

系统需返回：

```json
{
  "challenge": "random_string_xxx"
}
```

### 签名验证

请求头 `x-douyin-signature` 包含签名，算法为 `SHA1(clientSecret + body)`。

### 线索创建逻辑

- `externalUserId` = `dy_{from_user_id}`
- `source` = `"douyin"`
- `status` = `"new_lead"`

## 5. 微信公众号

### 接入方式

**被动回调模式**：微信公众号平台将用户消息推送到本系统的 Webhook 地址。支持明文模式和安全模式（AES加密）。

### 后台配置项

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `appId` | 可选 | 公众号AppID（用于获取用户昵称等） |
| `appSecret` | 可选 | 公众号AppSecret |
| `token` | 必填 | 消息验证Token |
| `encodingAesKey` | 可选 | 消息加解密密钥（安全模式必填） |

### Webhook 地址

```
GET/POST https://kefu.jq-industries.com/api/mp/webhook
```

- **GET**：微信后台配置服务器URL时的验证请求
- **POST**：接收用户消息推送，Body 为 XML 格式

### 配置步骤

1. 在系统"设置 → 平台消息 → 公众号"中填写 Token（和 EncodingAESKey）
2. 在微信公众平台 → 开发 → 基本配置中，设置服务器URL为 `https://kefu.jq-industries.com/api/mp/webhook`
3. 填入相同的 Token 和 EncodingAESKey，选择消息加解密方式

### 线索创建逻辑

- `externalUserId` = `mp_{openid}`
- `source` = `"mp_wechat"`
- `status` = `"new_lead"`

## 6. 视频号 / 微信小店客服

### 接入方式

**被动回调模式**：微信小店（视频号客服）将客服消息推送到本系统。消息格式为 JSON，强制安全模式（纯密文）。

### 后台配置项

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `appId` | 必填 | 微信小店的AppID |
| `appSecret` | 必填 | 微信小店的AppSecret |
| `token` | 必填 | 消息推送验证Token |
| `encodingAesKey` | 必填 | 消息加解密密钥 |

### Webhook 地址

```
GET/POST https://kefu.jq-industries.com/api/channels-shop/webhook
```

- **GET**：配置消息推送URL时的验证请求
- **POST**：接收客服消息推送，Body 为 JSON `{ "ToUserName": "...", "Encrypt": "..." }`

### 配置步骤

1. 在系统"设置 → 平台消息 → 视频号"中填写 AppID、AppSecret、Token、EncodingAESKey
2. 在微信小店后台 → 开发设置中配置消息推送URL

### 线索创建逻辑

- `externalUserId` = `channels_shop_{FromUserName}`
- `source` = `"channels_shop"`
- `status` = `"new_lead"`

## 7. 视频号私信（自定义脚本上报）

### 接入方式

**主动上报模式**：通过本地脚本或自动化工具，调用系统提供的 REST API 上报视频号私信消息。适用于无法直接配置 Webhook 的场景。

### 认证方式

请求头需携带 API Key：

```
X-API-Key: {CHANNELS_PRIVATE_API_KEY}
```

### 接口列表

#### 单条消息上报

```
POST https://kefu.jq-industries.com/api/channels-private/messages
```

**请求体：**

```json
{
  "nickname": "用户昵称",
  "content": "消息内容",
  "timestamp": 1714300200000,
  "openId": "用户openId（可选）",
  "msgType": "text",
  "mediaUrl": "https://...(图片/视频URL，可选)",
  "company": "公司名称（可选）",
  "phone": "电话号码（可选）"
}
```

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `nickname` | 必填 | 发送者昵称 |
| `content` | 必填 | 消息文本内容 |
| `timestamp` | 可选 | 消息时间戳（毫秒） |
| `openId` | 可选 | 用户唯一标识 |
| `msgType` | 可选 | 消息类型（text/image/video等） |
| `mediaUrl` | 可选 | 媒体文件URL |
| `company` | 可选 | 公司名称（用于客户记录） |
| `phone` | 可选 | 电话号码（用于客户记录） |

**响应：**

```json
{
  "success": true,
  "messageId": 123,
  "customerId": 456
}
```

#### 批量消息上报

```
POST https://kefu.jq-industries.com/api/channels-private/messages/batch
```

**请求体：**

```json
{
  "messages": [
    { "nickname": "用户A", "content": "消息1" },
    { "nickname": "用户B", "content": "消息2", "company": "XX公司", "phone": "13800138000" }
  ]
}
```

- 单次最多 **100 条**消息
- 每条消息的字段要求同单条上报

#### 健康检查

```
GET https://kefu.jq-industries.com/api/channels-private/health
```

需携带 `X-API-Key` 请求头。

### 线索创建逻辑

- `externalUserId` = `channels_shop_{openId}` 或 `channels_shop_private_{nickname}`
- `source` = `"channels_shop"`
- `status` = `"new_lead"`

## 8. 官网表单

### 接入方式

**Webhook 推送模式**：官网表单提交后，通过企业微信群机器人格式推送到本系统。支持三种表单类型：联系咨询、招聘咨询、资料下载申请。

### Webhook 地址

```
POST https://kefu.jq-industries.com/api/webhook/guiqiao
```

（兼容旧地址：`POST /api/website-lead/webhook`）

### 请求格式（推荐：企业微信 Markdown 格式）

```json
{
  "msgtype": "markdown",
  "markdown": {
    "content": "## 📨 新的联系咨询\n> 提交时间：2026/4/28 16:30:00\n**姓名：** 张三\n**公司：** 某某科技有限公司\n**邮箱：** zhangsan@example.com\n**电话：** 13800138000\n**主题：** 产品咨询\n**内容：** 我想了解你们的智能纤维传感产品"
  }
}
```

### 支持的表单类型

| 表单类型 | 标题关键词 | 典型字段 |
|---------|-----------|---------|
| 联系咨询 | `新的联系咨询` | 姓名、公司、邮箱、电话、主题、内容 |
| 招聘咨询 | `新的招聘咨询` | 姓名、邮箱、电话、应聘职位、工作经验、学历、留言 |
| 资料下载 | `新的资料下载申请` | 姓名、公司、邮箱、电话、职位、行业、资源 |

### Markdown 字段格式

每个字段一行，格式为：

```
**字段名：** 字段值
```

提交时间格式：

```
> 提交时间：2026/4/28 16:30:00
```

### 兼容旧格式（直接 JSON 字段）

```json
{
  "name": "张三",
  "email": "zhangsan@example.com",
  "phone": "13800138000",
  "company": "测试科技有限公司",
  "message": "我想了解你们的产品"
}
```

### 响应格式

```json
{ "errcode": 0, "errmsg": "ok" }
```

### 线索创建逻辑

- `externalUserId` = `website_{email}` 或 `website_{phone}` 或 `website_{name}`
- `source` = `"website"`
- `status` = `"info_collected"`（官网留资直接标记为已留资）
- 客户名称优先使用公司名，其次用姓名

### 去重逻辑

系统按以下优先级查找已有客户：
1. `externalUserId` 完全匹配
2. `email` 匹配
3. `phone` 匹配

如果找到已有客户，更新信息并添加跟进记录；否则创建新客户。

## 9. 邮箱

### 接入方式

**IMAP 拉取模式**：系统主动连接指定邮箱的 IMAP 服务器，定期拉取收件箱中的邮件并创建客户记录。**不是 Webhook 模式**，无需外部系统推送。

### 后台配置项

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `targetEmail` | 必填 | 要监控的邮箱地址 |
| `imapPassword` | 必填 | IMAP授权密码/应用专用密码 |
| `imapHost` | 可选 | IMAP服务器地址（默认 `imap.exmail.qq.com`） |
| `imapPort` | 可选 | IMAP端口（默认 `993`） |
| `smtpHost` | 可选 | SMTP服务器地址（回复邮件用） |
| `smtpPort` | 可选 | SMTP端口 |
| `smtpUser` | 可选 | SMTP用户名 |
| `smtpPassword` | 可选 | SMTP密码 |
| `senderName` | 可选 | 发件人显示名称 |

### 使用方式

1. 在系统"设置 → 平台消息 → 邮箱"中配置邮箱信息
2. 点击"同步邮件"按钮手动拉取，或等待系统定时自动同步
3. 新邮件会自动关联到已有客户（通过发件人邮箱匹配），或创建新客户记录

### 线索创建逻辑

- `source` = `"email"`
- 客户匹配基于发件人邮箱地址

## 10. 手动录入

### 接入方式

在系统界面中手动创建客户记录。

### 操作路径

系统登录后 → 客户管理 → 点击"添加客户"按钮

### 可填字段

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `name` | 必填 | 客户名称（建议使用公司名） |
| `company` | 可选 | 公司名称 |
| `phone` | 可选 | 电话号码 |
| `email` | 可选 | 邮箱地址 |
| `department` | 可选 | 分配事业部（精密/关怀/定制/实验室/销售中心/其他） |
| `assignee` | 可选 | 对接人 |
| `notes` | 可选 | 备注信息 |
| `source` | 可选 | 来源（默认为 `manual`） |

### tRPC 接口

```
trpc.customer.create.useMutation()
```

## 11. 电话

### 接入方式

**手动录入**：电话来源的客户通过手动创建，在来源字段选择"电话"。系统目前没有电话平台的自动化 Webhook 接口。

### 线索创建方式

在系统中手动添加客户时，将 `source` 设为 `"phone"` 即可。
